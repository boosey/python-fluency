Lesson 17 · Idiomatic Python

The Zen of Python, applied

Sixteen lessons taught you the idioms. This one shows you the single philosophy underneath them — so that "is this Pythonic?" becomes a question you can answer yourself.

Open any Python prompt and type one line:

import this

You'll get a 19-line poem — Tim Peters' Zen of Python, enshrined as PEP 20. It is real, importable code shipped in the standard library, and it is the closest thing the language has to a constitution. Nearly every idiom in this course is one of its aphorisms made concrete. Let's connect the load-bearing ones to what you already learned.

Five aphorisms, five idioms you already know

“Readability counts.” The whole point of a comprehension or an f-string is that the code reads like the sentence that describes it — [n * n for n in nums] says what you want, not the mechanics of gathering it (lessons 01 and 06).

“There should be one — and preferably only one — obvious way to do it.” That obvious way already exists for iterating: reach for enumerate, zip, and with instead of hand-rolling indices, parallel loops, or cleanup (lessons 05 and 09).

“Flat is better than nested.” A guard clause that returns early flattens a pyramid of if blocks; a single comprehension flattens a build-loop-append. When nesting climbs past two levels, the idiom is usually to step back out (lessons 01 and 03).

“Explicit is better than implicit.” Naming keyword arguments, unpacking into named targets, and avoiding clever magic all make intent visible at the call site — the reader shouldn't have to guess (lessons 02 and 08).

“Errors should never pass silently.” The Pythonic style is EAFP — try the thing and catch the specific exception — never a bare except: that swallows everything. Ask forgiveness, but only for the error you expected (lessons 03 and 07).

The unifying idea: "Pythonic" is not a style preference. It means the code lets the language carry the weight — the data model, the built-ins, the protocols — instead of porting a pattern from somewhere else.

Pythonic vs not — the highest-value swaps

If you remember nothing else, remember these five before/after pairs. Each one is a pattern you'd write in many languages, and the form Python prefers instead.

Ported pattern

# manual index loop
for i in range(len(items)):
    print(i, items[i])

Idiomatic Python

# enumerate hands you both
for i, item in enumerate(items):
    print(i, item)

Ported pattern

# compare length to zero
if len(items) > 0:
    handle(items)

Idiomatic Python

# empty is already falsy
if items:
    handle(items)

Ported pattern

# build, loop, append
out = []
for n in nums:
    out.append(n * n)

Idiomatic Python

# one expression
out = [n * n for n in nums]

Ported pattern

# open and close by hand
f = open(path)
data = f.read()
f.close()

Idiomatic Python

# with closes, even on error
with open(path) as f:
    data = f.read()

Ported pattern

# getters and setters
class Point:
    def get_x(self):
        return self._x

Idiomatic Python

# dataclass: plain attributes
from dataclasses import dataclass

# @dataclass
class Point:
    x: int; y: int

Plain attribute access stays plain; reach for @property only when you need computed or validated access — not by default (lessons 10 and 11).

The idiom checklist — print this

A page to keep beside your editor. Before you call a chunk of Python "done", scan it:

  • Looping by index? Use enumerate; for two sequences, zip.
  • Building a collection in a loop? Try a comprehension or a generator expression first.
  • Checking len(x) > 0 or x == None? Write if x: and if x is None:.
  • Acquiring a resource? A file, lock, or connection belongs in a with block.
  • Unpacking a tuple by index? Assign to named targets: x, y = point.
  • Looking before you leap? Prefer EAFP — try and catch the specific exception.
  • Writing getters/setters? Use a @dataclass; add @property only when access needs logic.
  • Concatenating strings in a loop? Collect and "".join(parts); format with an f-string.
  • Reaching for a default in a dict? Use .get, setdefault, or defaultdict.
  • Nesting more than two levels? Flatten with a guard clause or an early return.
