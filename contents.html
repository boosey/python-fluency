Idiomatic Python

For experienced programmers learning what makes Python Python — the distinctive syntax, idioms, and best practices, taught as the delta from other languages. Each lesson is short, has an embedded voice-over, and ends with a quick quiz.

How to use this course: work through the lessons in order — each builds on the last. Click ▶ Play narration at the top of any lesson to hear it read aloud (uses your browser's built-in speech — no install). Answer each quiz before scrolling to the feedback: recall is what makes idioms stick. Stuck on anything? Ask your teacher (me) directly.
  1. 01 Comprehensions The most recognisable line of Python you'll ever read — and the fastest way to start writing Python that looks native instead of ported.
  2. 02 Unpacking & multiple assignment Stop indexing sequences one element at a time. Python lets you name the pieces in a single line — and once it's automatic, half your “glue” code disappears.
  3. 03 Truthiness & boolean idioms Python objects already know if they're "empty" — lean on that and your conditionals read like English instead of like a port from another language.
  4. 04 Slicing One compact notation reads a whole region out of any sequence — master the half-open interval and slices stop being something you puzzle over and become something you skim.
  5. 05 Iteration & its built-ins Python has no C-style index loop. You iterate objects directly — and a handful of built-ins make almost every "I need the index too" case index-free.
  6. 06 Strings & f-strings One syntax for formatting, one idiom for building — f-strings and join replace the printf, concatenation, and StringBuilder habits you arrived with.
  7. 07 Dictionary idioms You already reach for a hash map without thinking. This lesson swaps a few ported habits for the Python forms that read as native.
  8. 08 Function arguments Python signatures are more flexible than almost any language you've used. Learn to read and write them, and you can follow real-world code — and APIs — without squinting.
  9. 09 Context managers One block, guaranteed cleanup. with ties a resource's whole lifetime to a scope — so it's released the instant you leave, exception or not.
  10. 10 The data model & dunder methods Python has no operator+ keyword and no interface to declare. You make your objects feel native by implementing the same named methods the language already calls on its builtins.
  11. 11 Decorators The @ line that sits above so many Python functions isn't magic — it's one tiny rule applied consistently. Learn the rule and you read and write decorators with confidence.
  12. 12 Generators & yield Write a function that produces a stream instead of returning a finished collection — lazy, resumable, and constant-memory, even over infinite data.
  13. 13 Dataclasses & named tuples Model plain data without the ceremony. One decorator writes the constructor, the printer, and the equality check — so you describe the shape and Python builds the machinery.
  14. 14 Exceptions & EAFP Most languages treat exceptions as a fire alarm. Python treats them as a normal way to ask "did that work?" — and embracing that makes your code shorter, faster, and more correct.
  15. 15 Type hints Annotations that document your intent and power your editor and type checker — without changing how a single line runs.
  16. 16 Structural pattern matching It looks like a switch, but it isn't one. match branches on the shape of your data and pulls values out as it goes — Python 3.10's most powerful reading skill.
  17. 17 The Zen of Python, applied Sixteen lessons taught you the idioms. This one shows you the single philosophy underneath them — so that "is this Pythonic?" becomes a question you can answer yourself.
