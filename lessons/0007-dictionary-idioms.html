Lesson 07 · Idiomatic Python

Dictionary idioms

You already reach for a hash map without thinking. This lesson swaps a few ported habits for the Python forms that read as native.

The dict is the most-used container in Python, so the difference between clumsy and fluent dict code shows up on nearly every screen you write. The win here is a small set of idioms you'll use daily.

Lookups with a fallback: .get()

Most languages

# check, then index
if key in d:
    val = d[key]
else:
    val = 0

Idiomatic Python

# one lookup, with a default
val = d.get(key, 0)

d.get(key) returns the value, or None when the key is absent — no KeyError. Pass a second argument to choose the default: d.get(key, 0). Reach for .get() any time a missing key is normal rather than exceptional.

Grouping: setdefault and defaultdict

To bucket items by a key, the one-line classic is setdefault: it inserts a fresh value the first time, then returns whatever is there.

# group words by their first letter
groups = {}
for w in words:
    groups.setdefault(w[0], []).append(w)

The cleaner form is collections.defaultdict, which manufactures the missing value automatically the moment you touch a new key:

from collections import defaultdict

groups = defaultdict(list)
for w in words:
    groups[w[0]].append(w)
The mental shift: you pass list (the function itself, not list()) as the factory. defaultdict calls it to make a default whenever a key is first accessed — so the append always has something to append to.

Counting: Counter

When the value you want is a count, skip the manual bookkeeping and use collections.Counter, a dict subclass built for tallying:

from collections import Counter

counts = Counter(words)        # {'the': 4, 'cat': 2, ...}
counts.most_common(3)            # top 3 as (item, count) pairs
counts["unseen"]              # 0 — never a KeyError

Iterating and membership

Iterate .items() to get key and value together, and test membership against the dict directly — key in d, never key in d.keys():

# key and value in one clean loop
for name, score in scores.items():
    print(name, score)

# membership: short and fast
if "alice" in scores:
    ...

Iterating a dict directly (for k in d) already yields keys, so d.keys() there is redundant. key in d tests keys too — and is faster than scanning d.keys().

Dict comprehensions and inverting

The comprehension shape from lesson 01 applies to dicts: curly braces, key: value, for each pair. That's all you need to invert a dict in one line:

# swap keys and values
inverse = {v: k for k, v in d.items()}

Inverting assumes the values are unique and hashable; duplicate values collapse, keeping the last one seen.

Merging dicts

To combine dicts, the spread form works everywhere; from Python 3.9 the | operator does the same with less noise. On overlap, the right-hand value wins:

# build a new merged dict
merged = {**a, **b}          # all versions
merged = a | b               # 3.9+, same result

# merge in place — update `a`
a |= b                       # 3.9+
Order is guaranteed: since Python 3.7, dicts preserve insertion order. Iteration, .items(), and merges all follow the order keys were first added — so output is predictable, and there's rarely a reason to reach for OrderedDict anymore.

Two for your pocket

val = d.pop(key, None)         # remove & return, default if absent
seen = dict.fromkeys(items, 0)   # {item: 0, ...} from a sequence of keys

d.pop(key, default) removes a key without risking a KeyError; dict.fromkeys(keys) builds a dict from a sequence of keys (all sharing one default value).
