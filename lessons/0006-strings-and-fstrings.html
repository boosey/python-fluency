Welcome to lesson six: strings and f-strings. You already format strings in whatever language you came from, so this lesson is about the one Python way that has quietly replaced all the others. That way is the f-string: put an f before the opening quote, and any expression inside curly braces is evaluated and dropped right into the text. So f, quote, name is age, with name in braces and age in braces, reads almost exactly like the sentence it produces. The braces are not limited to bare variables; you can put any expression inside, like a method call or arithmetic. A small gift from Python 3.8 is the self-documenting form: write the expression, then an equals sign, still inside the braces, and you get both the expression text and its value. After the value you can add a colon and a format spec to control how it looks. Colon dot two f rounds a float to two decimal places; colon comma adds thousands separators; colon greater-than ten right-aligns in a field of width ten; colon zero three d pads an integer to three digits with leading zeros. Now the second big idea: building a string from many pieces. In many languages you reach for a StringBuilder, or you append to a string inside a loop with plus-equals. In Python you collect the pieces in a list and then call join on the separator, like empty-quote dot join of parts. Join is idiomatic because Python strings are immutable, so plus-equals in a loop quietly creates a brand new string every iteration. Immutability is the theme of the whole type: every string method returns a new string and never changes the original. Strip, split, replace, starts-with, and lower all hand you a fresh value, so you assign the result rather than expecting a mutation. To test whether one string contains another, you do not call a method at all; you just write the in operator. Splitlines breaks text into a list of lines, and triple-quoted strings let you write blocks that span several lines. For regular expressions and Windows paths, reach for a raw string by prefixing the quote with r, which turns off backslash escapes. A quick word on history: you may still meet percent-formatting and the dot-format method, but for new code prefer f-strings, which are clearer and faster. There is no printf and no StringBuilder here; the f-string and join cover almost everything. Scroll down, take the quick checks, and replay this narration whenever you like. Lesson 06 · Idiomatic Python

Strings & f-strings

One syntax for formatting, one idiom for building — f-strings and join replace the printf, concatenation, and StringBuilder habits you arrived with.

You already know how to stitch text together. The win here is narrow: make the f-string your reflex for formatting, and "".join(...) your reflex for building, so you stop reaching for the patterns Python doesn't want.

The f-string: expressions inside the text

Other formatting styles

# positional / printf-ish / .format
"%s is %d" % (name, age)
"{} is {}".format(name, age)

Idiomatic Python

# the value sits where you read it
f"{name} is {age}"

The f prefix turns the literal into a template: anything inside { } is a real Python expression, evaluated in place. That means method calls and arithmetic live right where the value appears:

f"{name.upper()} scored {score * 100:.0f}%"
f"total: {price + tax}"
Self-documenting (3.8+): add an = after the expression and the f-string prints both the source text and its value — perfect for quick debugging: 
x = 42
print(f"{x=}")   # x=42

Format specs: the part after the colon

Inside the braces, a : introduces a format spec that controls presentation without touching the value:

f"{x:.2f}"    # 3.14159  -> '3.14'   (2 decimal places)
f"{n:,}"      # 1000000  -> '1,000,000' (thousands)
f"{p:>10}"    # 'hi'     -> '        hi' (right-align, width 10)
f"{i:03d}"    # 7        -> '007'   (zero-pad to 3 digits)

You can combine them — alignment, width, precision, and grouping all stack in one spec, e.g. f"{value:>12,.2f}".

Building strings: join, not +=

The habit to drop

# new string every iteration
out = ""
for p in parts:
    out += p + ","

Idiomatic Python

# one allocation, separator up front
out = ",".join(parts)

There is no StringBuilder in Python, and you don't need one. Because strings are immutable, += in a loop quietly allocates a fresh string each pass. str.join walks the pieces once and builds the result in a single pass — call it on the separator: "".join(parts) glues with nothing between.

The theme — immutability: every string method returns a new string and never mutates in place. So you assign the result; you don't expect the original to change.

Methods return new strings

s = "  Hello, World  "
s.strip()              # 'Hello, World'  (s is unchanged)
s.lower()              # '  hello, world  '
s.replace("l", "L")   # '  HeLLo, WorLd  '
"a,b,c".split(",")    # ['a', 'b', 'c']
"file.py".startswith("file")  # True

To test for a substring, don't call a method — use the in operator, which reads like English:

if "@" in address:       # substring test
    ...
for line in text.splitlines():  # list of lines, no trailing \n
    ...

Multi-line and raw strings

# triple quotes span lines verbatim
msg = """Dear user,
Your order shipped.
"""

# raw string: backslashes are literal — for regex and paths
pattern = r"\d+\.\d+"
path    = r"C:\Users\me"

An r prefix turns off backslash escapes, so \d stays two characters instead of becoming an escape — exactly what regex and Windows paths want. (You can combine: rf"..." is a raw f-string.)

Old code uses % and .format(); you should be able to read them, but reach for f-strings in new code — they're clearer and faster.
