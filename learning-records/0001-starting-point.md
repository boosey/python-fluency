# Starting point: experienced programmer, zero assumed Python idiom

The learner is an experienced programmer in other languages and explicitly wants
the *delta* — Python's distinctive syntax, idioms, control structures, and best
practices — not programming fundamentals. Mission is general fluency: reading and
writing idiomatic Python comfortably.

Implication: teach every idiom by contrast with how other languages do it, and
start from the most visually distinctive, highest-frequency idioms (comprehensions,
unpacking, iteration protocol) before moving to the data model (dunder methods,
context managers, decorators). No need to justify *that* a feature exists — focus
on *why the Pythonic form is preferred* and *how to read it fast*.
