# Mission: Idiomatic Python for an experienced programmer

## Why
I already know how to program. I want to read and write *idiomatic* Python
fluently — to recognise the constructs that make Python distinctive (not just
"the same code with different syntax"), so that Python code reads naturally to
me and the Python I write looks like a native wrote it, not a translation from
another language.

## Success looks like
- I can read idiomatic Python (comprehensions, generators, unpacking, context
  managers, dunder methods, decorators) at a glance, without slowing down.
- When I write Python, I reach for the Pythonic construct first instead of
  porting a pattern from another language (no manual index loops, no getters/
  setters, no `StringBuilder`).
- I can pass a Python code review on style and idiom, and explain *why* the
  idiomatic form is preferred.

## Constraints
- Experienced programmer — skip "what is a variable / a loop". Teach the delta
  between Python and other languages, not programming fundamentals.
- Short lessons, one tangible win each. Voice-over narration on every lesson.

## Out of scope (for now)
- Specific frameworks (Django, FastAPI, pandas, NumPy). Language and stdlib idiom first.
- Packaging, virtualenvs, tooling. Revisit once the language idioms are solid.
- Async/concurrency. A later mission once the core idioms are fluent.
